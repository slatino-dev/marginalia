# DESIGN BRIEF: Marginalia (Design Loop step 1)

## Purpose
A reading instrument that proves its answers. The demo page must make a hiring manager involuntarily click a margin note within 90 seconds, then find, behind that click, the exact source passage and the full retrieval provenance that produced it. The screen has to read as a real system with real telemetry and real taste, not a chat bubble with a sources accordion.

## Audience
AI-infrastructure hiring managers and senior engineers giving the portfolio 90 seconds (primary), and engineers who arrive on the eval tab or a shared-answer permalink evaluating the RAG architecture (secondary). Both must instantly see: honest retrieval, auditable citations, published metrics.

## ONE named direction: "Signal Path" (lab-wide language v2)
The professional routing-hardware aesthetic of the family: dense-but-calm dark surfaces, precise 1px rules and connector geometry, live telemetry (candidate counts, per-lane ranks, fusion and rerank scores, TTFT) treated as first-class design elements with tabular-nums everywhere, one display face with genuine character + one engineered body/mono pairing chosen at Design Loop step 3, OKLCH token scales, Phosphor icons (Regular UI / Fill active / Duotone illustration). Marginalia consumes the shared Signal Path token base and overrides exactly two things:

- **Accent: "Rubric" vermilion**, OKLCH hue ~40 (working center `oklch(0.66 0.19 40)`), from rubrication, the red-ink margin annotation that IS marginalia. The patch-cable metaphor becomes the citation thread: a vermilion cable runs from the claim in the answer to its source in the margin. Error semantics shift cooler and darker (hue ~20 to 25) so rubric never reads as failure. Because rubric (hue ~40) and error (hue ~20 to 25) sit only ~15 to 20 degrees apart in the same warm family, **hue is NOT a safe separator**: the citation thread must be distinguished from an error highlight primarily by lightness and chroma, and a deuteranopia + protanopia simulation on accent-vs-error is a required check at Design Loop step 5. Resolved in the lab-wide coherence pass (2026-07-08): Crucible took molten ember (~55) and Demiurge sunrise-amber (~80), while **Conduit vacated the warm arc for signal cyan (~205)**, so the warm family is now just Marginalia (~40), Crucible (~55), and Demiurge (~80), each ≥15 to 25 degrees apart. Rubric at ~40 stays distinct but adjacent to ember; the deuteranopia + protanopia check on accent-vs-error (hue ~20 to 25) below remains the load-bearing separation test, since lightness and chroma, not hue, carry it.
- **Display flourish: the annotated page.** Answers are typeset as a manuscript-quality reading column (measure 60 to 70ch) with citations as LIVE MARGIN NOTES in the right margin at >=1024px, collapsing to inline popovers below. This is the name made visible. **Mobile is a designed activation path, not a fallback**: at 390px the recruiter still reaches `citation_audit_opened` — tapping an inline citation opens a bottom-sheet audit panel carrying the same provenance content (source passage highlighted, document + page anchor, that chunk's full retrieval trace) as the desktop margin note. Verified at the mandatory 390px screenshot pass.

## The hero moment (the ONE orchestrated motion)
The retrieval trace draws itself as a signal path when a question runs: the query splits into sub-query lanes, the BM25 and dense channels light up with their candidate counts, the paths converge through fusion and reranking nodes, and the surviving passages snap into the margin as the answer streams token by token with a caret. Transform and opacity only; all timing from motion tokens; `prefers-reduced-motion` swaps the whole thing to an instant static route diagram. Secondary micro-moment, reusing the same motion vocabulary rather than adding a new one: opening a citation sweeps the highlight across the source passage in the audit panel.

## Alternates (if Signal Path fails review at Design Loop step 6)
1. **Blueprint** — drafting-table linework, annotation-driven labels, engineering-drawing calm; the margin notes read as draftsman's callouts and the retrieval trace as a schematic.
2. **Terminal Luxe** — phosphor-heritage, extreme typographic restraint, zero chrome; the answer is a lit reading column and the trace is monospace telemetry.

## Constraints
Dark-first with a real token-level light remap. Five designed states per view, always: loading (layout-matched skeleton, zero CLS), empty, error, degraded, populated. The empty state IS onboarding: preset questions presented as patch-cable jacks waiting to be connected. Degraded mode is a designed state with honest copy ("running retrieval-only, generation paused to protect the shared model budget"), never an error toast. Eval-tab charts follow the dataviz rules: zero-baseline bars, direct labels over legends, the insight as the title (for example "Hybrid beats dense-only by N points on this corpus"), colorblind-safe categorical palette from tokens, tabular-nums right-aligned in tables (TanStack Table). WCAG AA; designed focus outlines (never removed); LCP <=2.5s, INP <=200ms, CLS <=0.1. Anti-slop bans in full: no Inter/Roboto/Arial display type, no purple-indigo gradients, no three identical cards in a row, no uniform-radius-everywhere, no emoji as UI icons, no scattered micro-animations, no fake-precision stats (every number on screen resolves to a real D1 row).

## Differentiation
Every RAG demo is a chat bubble with a sources accordion. An annotated manuscript page, with a live retrieval schematic drawing itself and public eval charts one tab away, is the screenshot that travels. Design the answer view to be screenshotted: the margin note, the vermilion citation thread, and the honest telemetry are the composition, not decoration.

**How this differs visually from Conduit (sibling, also a live-trace demo):** both are query-trace-into-cited-answer demos, so the difference must be experiential, not just prose. Marginalia's hero is a **vertical manuscript reading column with a vermilion citation thread running into a right-margin note gutter**, over a **BM25↔dense fusion + rerank schematic** — a reading instrument. Conduit's is a **horizontal rack of cable segments (one per EDGAR tool call) in signal cyan** with a federal rate-budget gauge — data infrastructure. Different color (vermilion vs cyan), different layout axis (vertical page + margin vs horizontal rack), different hero object (margin note vs rate-budget gauge). The backend split (hybrid-retrieval fusion vs deterministic tool-fetch) is made *visible* through these, not left as invisible architecture. Verified at the 390/768/1440px screenshot pass: the two demos must not travel as lookalike images.
